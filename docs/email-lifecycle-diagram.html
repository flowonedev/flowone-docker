warning UID Change is Critical

  • IMAP assigns a new UID when a message moves between folders
  • The old UID is permanently invalidated in the source folder
  • getLastMoveNewUid() captures the server-assigned UID
  • ConversationService is updated with the new UID so threads remain intact
  • If new UID capture fails, conversation reconciliation fixes it later (2-min cycle)

shield Safety Nets

  • Reconciliation timer: Every 2 minutes, re-fetches folders + current folder as a safety net for missed events
  • Event versioning: lastEventVersion persisted in IndexedDB to detect gaps
  • Duplicate guard: processedEventIds Set prevents handling the same event twice
  • Visibility refresh: When tab becomes visible after being hidden, triggers a full refresh
  • Reconnect handler: On WebSocket reconnect, refreshes all data to catch up

speed Performance Guards

  • Debounced fetches: Folder and message fetches are debounced (300ms default) to prevent API flood
  • Conditional auto-read: message() only sets \Seen if message is actually unread
  • Debounced rule evaluation: Board Pro email rules batched on 300ms timer
  • Duplicate Sieve guard: Block-sender filter checks for existing rules before appending
